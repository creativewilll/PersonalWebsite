# H1 must restate title with a buyer hook (no clickbait)

**Lead paragraph (2–3 sentences):** Answer *who*, *what outcome*, and *why now*. Mention orchestration platform (e.g., n8n) and system of record if applicable.

## Who is this for?

Bullets: ICP, team size signal, tech maturity. **Forbidden:** generic “every business.”

## What breaks when you do this manually?

Quantify *types* of failure (latency, duplicate records, missed follow-up), not fabricated dollar claims.

## What ships (deliverable checklist)

Numbered list: workflows, credentials pattern, docs, handoff call scope.

## Architecture at a glance

| Layer | Responsibility | Typical tools |
|-------|----------------|---------------|
| Trigger | ... | ... |
| Orchestration | ... | n8n |
| Intelligence | ... | LLM / rules |
| System of record | ... | ... |
| Notification | ... | ... |

## End-to-end flow (node-agnostic)

5–9 numbered steps from trigger to terminal action. Use **bold** for decision points and failure branches.

## Integrations and credentials

How OAuth vs API keys vs webhooks are scoped; rotation; least privilege.

## AI usage: where models help—and where they hurt

- **Use LLMs for:** classification, summarization, drafting, semantic routing.
- **Use code/rules for:** money, scheduling, dedupe keys, authorization.

## Reliability and governance

Retries, idempotency, dead-letter pattern, PII handling, audit log idea.

## Security and privacy

Data residency, what leaves the boundary, redaction, human approval gates.

## Deployment options

Self-hosted n8n vs cloud; staging; secrets; monitoring.

## Buying models (without BS)

Retainer, one-time build, template license—generic framing only.

## Manual vs automated (comparison)

Short table: speed, error rate class, cost structure (time vs infra)—no fake ROI.
